# クライアント情報チェックリスト

LINE公式アカウント連携と入退室通知システムの開発に必要な情報をまとめています。

---

## 📋 必須情報

### 1. LINE公式アカウント関連

#### ✅ LINE公式アカウントの有無
- [ ] **既にLINE公式アカウントを持っているか？**
  - ✅ はい → アカウント情報を提供してもらう
  - ❌ いいえ → 新規作成が必要（開発者側で作成可能）

#### ✅ LINE Developersアカウント
- [ ] **LINE Developersコンソールへのアクセス権限**
  - クライアントのLINE Business IDアカウント情報
  - または、開発者を管理者として追加してもらう

#### ✅ 必要な認証情報（開発者が取得）
以下は開発者がLINE Developers Consoleから取得します：
- [ ] **チャネルアクセストークン**（LINE Messaging API）
- [ ] **チャネルシークレット**（Webhook検証用）
- [ ] **QRコード**（親御さんへの配布用）

**取得方法:**
1. LINE Developers Console (https://developers.line.biz/console/)
2. 該当のチャネルを選択
3. 「Messaging API設定」タブから取得

### 5. 通知内容のカスタマイズ要件

#### ✅ 通知メッセージの内容

**現在のデフォルト:**
```
太郎さんが入室しました。
時刻: 2026/01/06 15:30
```

**カスタマイズ希望:**
- [ ] **教室名を含めるか？**
  - 例: 「【〇〇塾】太郎さんが入室しました。」

- [ ] **住所や教室情報を含めるか？**
  - 例: 「場所: 本校（東京都渋谷区...）」

- [ ] **ポイント情報を含めるか？**
  - 例: 「獲得ポイント: +10pt（合計: 120pt）」

- [ ] **次回の予定を含めるか？**
  - 例: 「次回: 1/8（水） 16:00〜」

- [ ] **その他追加したい情報**: ___________________________

#### ✅ 通知タイミング
- [ ] **入室時に通知**: ✅ / ❌
- [ ] **退室時に通知**: ✅ / ❌
- [ ] **自動退室時に通知**: ✅ / ❌
- [ ] **遅刻・早退時に通知**: ✅ / ❌

#### ✅ 通知先の設定
- [ ] **両親に通知する？**
  - [ ] 母親のみ
  - [ ] 父親のみ
  - [ ] 両方
  - [ ] 主たる連絡先のみ

## 📊 情報提供の優先順位

### 🔴 最優先（これがないと開発不可）
1. LINE公式アカウント（または作成の許可）
2. 生徒リスト（最低限: 名前・学年）
3. 親御さんの連絡先（最低限: 氏名・電話番号）
4. 通知内容の基本方針

### 🟡 重要（早めに確認したい）
5. NFCカードの準備状況
6. 入退室の運用ルール
7. セキュリティ・プライバシー方針

### 🟢 追加要件（後から調整可能）
8. 通知メッセージのカスタマイズ
9. ポイントシステムの詳細
10. 専用ドメイン・デザイン

## 🎯 次のステップ

### クライアントから情報を受領後:

1. **LINE公式アカウントの設定**（1日）
   - チャネル作成
   - チャネルアクセストークン取得
   - Webhook設定

2. **生徒・親御さんデータの投入**（1〜2日）
   - データベースへのインポート
   - NFCカードとの紐付け

3. **テスト運用**（3〜5日）
   - テストユーザーで動作確認
   - 親御さんへの通知テスト
   - クライアントと一緒に検証

4. **本番運用開始**
   - 親御さんへの案内配布
   - LINE友だち追加の促進
   - 運用開始 🎉

## 💡 Tips

### 情報収集のコツ:

1. **段階的に情報収集**
   - 最初は最優先情報のみ
   - 開発を進めながら詳細を詰める

2. **クライアントの負担を減らす**
   - 選択式の質問を多用
   - サンプル・例を提示

3. **実際のデータでテスト**
   - テスト用の実データ（1〜2件）を早めに取得
   - 実際の運用イメージを共有

4. **柔軟に対応**
   - 後からカスタマイズ可能な設計
   - 運用しながら改善
